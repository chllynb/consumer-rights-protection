# Complaint Channels For Mainland China Consumer Disputes

Use this file to choose the fastest practical complaint path. Local rules and interfaces can change, so the latest official entry page should prevail.

## Channel Selection

### 12315

Best for:

- False advertising
- Product quality issues
- Unfair terms
- Refund refusals
- Food issues
- General merchant-platform consumer disputes

Typical use:

- Choose the actual business entity when possible.
- If the platform collected payment or controlled the transaction flow, the platform entity may also matter.
- Upload concise evidence rather than every screenshot at once.

### 12345

Best for:

- The user does not know the correct regulator
- The case spans multiple departments
- The user already filed elsewhere but needs transfer or督办

Typical use:

- Good backup path when the user is unsure about the specific bureau.
- Useful for follow-up when a prior complaint is stalled.

### Platform internal complaint

Best for:

- Immediate refund or cancellation requests
- Building a record that the user already tried to resolve the issue

Typical use:

- Ask for a ticket number or agent ID.
- Keep the message factual and short.

### Public-pressure channels

Examples:

- Black Cat Complaint
- Social media posts with evidence comparisons

Use only as support, not as a substitute for official complaint channels.

## Common Routing By Scenario

| Scenario | First Choice | Backup | Notes |
| :--- | :--- | :--- | :--- |
| Hotel / OTA booking dispute | Platform complaint + 12315 | 12345 | If tourism elements are strong, local culture-and-tourism authorities may also matter. |
| Online shopping quality issue | Platform complaint + 12315 | 12345 | Focus on order, listing, and delivery evidence. |
| Food safety at restaurant | 12315 | 12345 | Preserve food, packaging, receipt, and medical proof. |
| Prepaid card store closure | 12315 or 12345 | Police or lawyer if fraud signs are serious | Multiple victims strengthen the public-interest angle. |
| Courier loss or delay | Merchant/platform first | Postal regulator route if needed | Keep tracking and delivery records. |

## Complaint Draft Skeleton

Use this structure for 12315 or 12345:

```text
投诉人于[日期]通过[平台/门店]购买[商品/服务]，订单号[XXX]，金额[XXX]元。

问题概述：
[用 2-4 句话说明承诺内容、实际情况、商家拒绝处理的情况]

已沟通情况：
[何时联系商家/平台，对方如何回复]

诉求：
1. [退款/退费/换货/赔偿]
2. [要求核查虚假宣传、格式条款等问题]

证据：
[订单截图、支付记录、商品或现场照片、聊天记录、录音、规则截图等]
```

## Practical Notes

- Prefer the company name from the payment record, invoice, order detail, or merchant registration page.
- If the user does not know the entity name, start with the platform name and note that the exact operating entity is being verified.
- Use short attachments with clear filenames if the platform supports uploads.
- For repeated refusal, mention the dates of prior contacts.
